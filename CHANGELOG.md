# Changelog

All notable changes to Lirox are documented here.

## [1.0.0] — 2024

### 🎉 First Stable Release

This release marks Lirox's transition from beta to a production-ready v1.0.0.

### Added
- **Code Inspection Engine** — agents can now READ, ANALYZE, and SELF-FIX code
  - `lirox/tools/code_reader.py` — safe code file reading with language detection
  - `lirox/tools/code_analyzer.py` — LLM-powered code analysis (syntax, security, logic)
  - `lirox/tools/code_executor.py` — safe Python execution for validation
  - `lirox/agents/code/code_inspector.py` — full inspection pipeline (read → analyze → fix)
- **Specialized agent subdirectories** with isolated memory per agent
  - `lirox/agents/code/` — CodeAgent with built-in code inspection
  - `lirox/agents/chat/` — ChatAgent for general conversation
  - `lirox/agents/finance/` — FinanceAgent for market and financial queries
  - `lirox/agents/browser/` — BrowserAgent for web research
  - `lirox/agents/research/` — ResearchAgent for deep analysis
- **Enhanced memory isolation** — `lirox/memory/memory_bank.py` with fully isolated per-agent KV store
- **Session management** — `lirox/memory/session_manager.py` singleton with session lifecycle management
- Type hints throughout all new modules
- Comprehensive docstrings on all new classes and functions
- Structured logging in all new modules

### Changed
- `APP_VERSION` in `lirox/config.py` bumped from `0.5.0` → `1.0.0`
- `__version__` in `lirox/__init__.py` bumped from `1.0.0b1` → `1.0.0`
- `version` in `pyproject.toml` bumped from `3.0.0` → `1.0.0`
- `lirox/main.py` module docstring and argparse description updated to v1.0.0
- `lirox/ui/display.py` module docstring updated to v1.0.0
- `README.md` heading updated to v1.0.0

### Fixed
- Memory isolation between agents — each agent now has a completely separate `MemoryManager` instance keyed by agent name
- Code generated by agents is now validated before being shown to the user

### Security
- Code execution in `CodeExecutor` uses `subprocess` without `shell=True`, preventing shell injection
- `CodeReader` enforces `SAFE_DIRS` path validation before reading any file

## [0.5.0] — beta

Initial public beta release with:
- Mind Agent architecture
- Multi-provider LLM support (Groq, OpenAI, Anthropic, Gemini, Ollama, etc.)
- Desktop control (file ops, terminal, screen capture)
- 3-tier memory system
- Sub-agent registry and skill system
- Self-improvement protocol (`/improve`)
